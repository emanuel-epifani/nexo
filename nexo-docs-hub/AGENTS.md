## üèóÔ∏è Project Structure & Organization

### Key Directories
- **`src/components/ui/`**: **READ-ONLYish**. Contains generic, reusable UI primitives (Buttons, Inputs, Cards) copied from shadcn/ui.
  - *Rule*: Do not modify these unless you are changing the global design system. If you need a new primitive (e.g., Accordion), "install" it by creating the standard shadcn component file here.
- **`src/components/`**: Feature-specific components (e.g., `DocsSidebar`, `CodeBlock`, `DocsHeader`).
- **`src/pages/docs/`**: Documentation content pages. Each file corresponds to a topic.
- **`src/layouts/`**: Layout wrappers. `DocsLayout.tsx` controls the sidebar/header structure for docs.
- **`src/lib/`**: Utilities. `utils.ts` contains the critical `cn()` helper.

### Import Convention
- Always use the `@/` alias for imports from `src/`.
  - ‚úÖ `import { Button } from "@/components/ui/button"`
  - ‚ùå `import { Button } from "../../components/ui/button"`

## üé® UI & Styling Guidelines

### 1. The `shadcn/ui` Pattern
This project does not use a massive UI library import. It uses "owned" components.
- **Usage**: Always check `src/components/ui/` for existing primitives before building custom ones.
- **Composition**: Build complex feature components by composing these primitives.

### 2. Tailwind & Semantic Colors
**NEVER hardcode hex values** (e.g., `#ff5500`). This breaks dark mode and theming.
Use the semantic utility classes defined in `tailwind.config.ts` and `index.css`:
- **Backgrounds**: `bg-background`, `bg-card`, `bg-muted`, `bg-primary`
- **Text**: `text-foreground`, `text-muted-foreground`, `text-primary-foreground`
- **Borders**: `border-border`, `border-input`
- **Interactive**: `hover:bg-accent`, `hover:text-accent-foreground`

### 3. Class Merging
Always use the `cn()` utility when accepting a `className` prop or merging conditional classes.
```tsx
import { cn } from "@/lib/utils";

export function MyComponent({ className }: { className?: string }) {
  return <div className={cn("bg-primary p-4", className)}>Content</div>;
}
```

### 4. Typography & Prose
For documentation content (in `src/pages/docs/`), do NOT style every paragraph manually.
- **Wrapper**: Wrap the content in `<div className="docs-prose">`.
- **Headings**: Use `<h1>`, `<h2>`, `<h3>`. The CSS in `index.css` targets these tags inside `.docs-prose` to apply correct font sizes and spacing.
- **Lists**: Use `<ul>` and `<li>`.
- **Code**: Use the `<CodeBlock />` component, not raw `<pre>`.

## üìù Workflow: Adding New Documentation

To add a new documentation page (e.g., "Transactions"):

1.  **Create File**: Create `src/pages/docs/Transactions.tsx`.
    - Template:
      ```tsx
      import CodeBlock from "@/components/CodeBlock";

      const Transactions = () => {
        return (
          <div className="docs-prose">
            <h1>Transactions</h1>
            <p>Explanation here...</p>
            <CodeBlock code={`// example code`} />
          </div>
        );
      };
      export default Transactions;
      ```
2.  **Add Route**: In `src/App.tsx`, import the page and add a `<Route>` under the `/docs` parent route.
3.  **Update Sidebar**: In `src/components/DocsSidebar.tsx`, add an item to the `sections` array with the correct path and an icon from `lucide-react`.

## üß© Code Patterns

### Components
- Functional components with `const` syntax.
- Props should be typed with `interface`.
- Default exports for Pages and Layouts. Named exports are acceptable for small utilities but consistency prefers Default for components.

### Icons
- Use `lucide-react`.
- Import individual icons: `import { Check, Copy } from "lucide-react";`

### State Management
- Use `useState` for local UI state (e.g., sidebar toggle, copy button feedback).
- Use `useEffect` sparingly.
- `tanstack/query` is available if fetching remote data is needed in the future.
